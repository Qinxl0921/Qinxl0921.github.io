OpenFeign

2024/12/29 微服务

一、OpenFeign基础使用

1、在pom.xml里面引入相关依赖

引入OpenFeign和负载均衡组件。

<!--openFeign-->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>
<!--负载均衡器-->
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>

2、在启动类中，使用@EnableFeignClients注解，启用OpenFeign功能

@EnableFeignClients
@SpringBootApplication
public class CartApplication {...}

3、编写Feign客户端

需要在类上加注解@FeignClient(value = “service_name”)，表明服务的名字。

在方法上定义请求方式，请求路径，返回值类型等等。

@FeignClient(value = "service_name")
public interface ItemClient{
    @GetMapping("/user")
    List<T> queryItemByIds(@RequestParam("ids") Collection<Long> ids);
}

4、使用FeignClient，实现远程调用

List<T> lists = itemClient.queryItemByIds(List.of(1,2,3));

二、连接池

原始的OpenFeign底层发http请求用的是jdk里面自带的HttpURLCollection，不带连接池，效率比较低。为了提高性能，引入okhttp连接池。

1、引入OKHttp依赖

<!--okhttp的依赖-->
<dependency>
    <groupId>io.github.openfeign</groupId>
    <artifactId>feign-okhttp</artifactId>
</dependency>

2、启动okhttp

在application.yml中启动okhttp

feign:
  okhttp:
    enabled: true

三、优雅地使用OpenFeign

每个微服务在使用OpenFeign时都需要编写Feign客户端，代码重复编写，麻烦！

采用一种新的架构，在各个微服务平级上新建一个api模块，里面定义三个包：

  1. client：里面存放整个项目中每一个微服务想要暴露的客户端。
  2. dto：存放所有微服务的需要公共访问的dto。
  3. config：存放OpenFeign的一些配置，比如日志配置等。

每一个微服务需要使用Feign客户端时，引用api模块的依赖即可。

四、OpenFeign日志输出

OpenFeign只会在FeignClient所在包的日志级别为DEBUG时，才会输出日志。其日志级别有4级:

  1. NONE：不记录任何日志信息，默认值。
  2. BASIC：仅记录请求的方法，URL以及响应状态码和执行时间。
  3. HEADERS：在BASIC基础上，记录了请求和响应头信息。
  4. FULL：记录所有请求和响应信息。

自定义OpenFeign的日志级别，需要声明一个类型为Logger.Level的Bean，在其中定义日志级别：

public class DefaultFeignConfig{
    @Bean
    public Logger.Level feignLogLevel(){
        return Logger.Level.FULL;
    }
}

如果想要配置某一个FeignClient的日志，可以在该客户端中@FeignClient注解中声明：

@FeignClient(value = "item-service", configuration = DefaultFeignConfig.class)

如果想要全局配置，让所有的FeignClient客户端都按照这个日志配置，则需要在微服务的启动类中@EnableFeignClients注解中声明：

@EnableFeignClients(defaultConfiguration = DefaultFeignConfig.class)
